1 Introduction

Species distribution modeling (SDM) relates species occurrence data to environmental predictors with the aim of estimating habitat suitability across space.

This document provides a minimal, reproducible SDM workflow in \(R\) using the sdm package (Naimi and Araujo 2016) version 1.2.67.

1.1 Loading required library

#install.packages('sdm') # In case you do not have it installed already
library(sdm)

1.2 Reading in the occurrence data

We will use data which comes with the package as that would be the easiest way to begin using the package. This data has presence (1) and absence (0) records of a virtual species. We will read it into \(R\) using the vect function from terra package (Abatzoglou et al. 2018) which is loaded whenever you load sdm.

sp <- vect(system.file("external/species.shp", package = "sdm"))

1.3 Reading in predictor variables

Apart from the occurrence records, we also need to load the predictor variables. There are four raster layers which come with the package and we will load them alongside the occurrence records to help in building the model. These are also often referred to as response variables. We will use rast function also from terra package (Abatzoglou et al. 2018).

preds <- rast(list.files(system.file("external", package = "sdm"), pattern = ".asc$", full.names = TRUE))

1.4 Visualize the data

It is important to visualize the output so that you confirm how the occurrence records, both presence and absence, are distributed over the environmental variable(s), (see Figure 1).

plot(preds$vegetation)
plot(sp[sp$Occurrence == 1, ], col = "blue", add = TRUE)
plot(sp[sp$Occurrence == 0, ], col = "red", add = TRUE)

1.5 Build the sdm data object

Once you are satisfied with the distribution of the occurrence records, the next step is to create an \(sdmData\) object. This is taking both the occurrence data and predictor variables.

d <- sdmData(
  formula = Occurrence ~.,
  train = sp,
  predictors = preds
)
d

The \(d\) object tells us information like number of species in our data set, the name of the species, number of features (predictor variables) and their names, type of the data such as presence-absence or presence-only or presence-background, whether we have separate/independent test data, number of records and whether they have coordinates. This is the data we supply to the sdm function to train the model.

1.6 Build the model

In building the sdm model, we provide our formula and the \(d\) object. We also provide the methods, replication method, fraction of the data for testing, and the number of runs. Here we can also provide other arguments like parallelSetting in case we want to use more cores in the computer and modelSettings to alter the default settings of each method.

m <- sdm(
  formula = Occurrence ~.,
  data = d,
  methods = "rf", # There are many methods, see getMethodNames()
  replication = "sub", # Can also use cv or boot.
  test.percent = 20,
  n = 10
)

To see which of the predictor variables was most important for defining the niche of the species, we can use getVarImp function as follows:

plot(getVarImp(m))

This shows us that vegetation, in this case, was the most important variable in explaining the niche of our species of interest.

1.8 Predict

Finally, the end game is to make prediction of habitat suitability of the species over the geographic space. Here we feed all the predictor variables to the model in order to get a continuous habitat suitability map as follows:

p <- predict(object = m, preds, mean = TRUE)
plot(p)

1.9 Binary Maps

After creating the habitat suitability map, we may be interested in actually showing where the species are present or absent. For that we need to binarize the suitability map so that we have presence (1) or absence (0). However, since the suitability map is continuous, it is difficult to strike a balance on where to pick the threshold. For this, we can go with a threshold that maximizes sensitivity and specificity. However, depending on the research objective, one may decide to use different threshold. For example, when trying to identify potential areas for conserving an endangered species, one may use lower thershold so that more area is considered present locations and demarcated for conservation, etc. Here we will go with the threshold that maximizes sensitivity and specificity.

pa <- pa(x = p, y = m, id = "ensemble", opt = 2)
plot(pa, col = c("darkgreen", "gray80"))
